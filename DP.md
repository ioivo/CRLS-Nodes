# 动态规划（Dynamic Programming）

## 概述

动态规划（DP）适用于具有以下两个性质的问题：
1. **最优子结构**：问题的最优解包含其子问题的最优解
2. **重叠子问题**：子问题被反复求解（区别于分治法）

DP 通常有两种实现方式：
- **带备忘的自顶向下法（Memoization）**：递归 + 缓存
- **自底向上法（Bottom-up）**：按子问题规模递增填表

---

## 14.1 钢条切割（Rod cutting）

### 问题描述

给定一根长度为 $n$ 英寸的钢条和一个价格表 $p_i$（$i=1,2,\ldots,n$），求切割方案使得**总收益最大**。

注意：如果 $p_n$ 足够大，最优解可能是一刀不切。

**示例**（$n=4$）：

| 长度 $i$ | 1 | 2 | 3 | 4 |
|:---:|:---:|:---:|:---:|:---:|
| 价格 $p_i$ | 1 | 5 | 8 | 9 |

长度为 4 的钢条共有 $2^{n-1}=8$ 种切法。最优方案：切成两段 2 英寸，收益 $5+5=10$。

### 递归解

设 $r_i$ 为长度为 $i$ 的钢条的最大收益，$r_0=0$。

$$r_n = \max_{1 \leq i \leq n}(p_i + r_{n-i})$$

**理解**：先切下长度为 $i$ 的一段（收益 $p_i$），剩下 $n-i$ 的长度按最优方式切（收益 $r_{n-i}$）。

### 朴素递归（指数级）❌

```
CUT-ROD(p, n)
1 if n == 0
2     return 0
3 q = -∞
4 for i = 1 to n
5     q = max(q, p[i] + CUT-ROD(p, n-i))
6 return q
```

时间复杂度：$T(n) = 1 + \sum_{i=1}^{n} T(n-i) = 2^n$ → 指数级！

### 带备忘的自顶向下 ✅

```
MEMOIZED-CUT-ROD(p, n)
1 let r[0..n] be a new array
2 for i = 0 to n
3     r[i] = -∞
4 return MEMOIZED-CUT-ROD-AUX(p, n, r)

MEMOIZED-CUT-ROD-AUX(p, n, r)
1 if r[n] ≥ 0
2     return r[n]        // 已计算过，直接返回
3 if n == 0
4     q = 0
5 else
6     q = -∞
7     for i = 1 to n
8         q = max(q, p[i] + MEMOIZED-CUT-ROD-AUX(p, n-i, r))
9 r[n] = q
10 return q
```

### 自底向上 ✅

```
BOTTOM-UP-CUT-ROD(p, n)
1 let r[0..n] be a new array
2 r[0] = 0
3 for j = 1 to n          // 按长度从小到大计算
4     q = -∞
5     for i = 1 to j
6         q = max(q, p[i] + r[j-i])
7     r[j] = q
8 return r[n]
```

时间复杂度：$\Theta(n^2)$（双重循环）

### 复杂度总结

| 方法 | 时间复杂度 | 空间复杂度 |
|:---|:---:|:---:|
| 朴素递归 | $\Theta(2^n)$ | - |
| 带备忘自顶向下 | $\Theta(n^2)$ | $\Theta(n)$ |
| 自底向上 | $\Theta(n^2)$ | $\Theta(n)$ |

## 14.2 矩阵链乘法（Matrix-Chain Multiplication）

### 问题描述

给定 $n$ 个矩阵的链 $\langle A_1, A_2, \ldots, A_n \rangle$，其中 $A_i$ 的维度为 $p_{i-1} \times p_i$。求一种**完全括号化**方案，使得计算乘积 $A_1A_2\cdots A_n$ 所需的**标量乘法次数最少**。

### 为什么括号化很重要？

**示例**：三个矩阵 $A_1(10\times100), A_2(100\times5), A_3(5\times50)$

| 方案 | 计算过程 | 总代价 |
|:---|:---|---:|
| $(A_1A_2)A_3$ | $10\times100\times5 + 10\times5\times50$ | **7,500** ✅ |
| $A_1(A_2A_3)$ | $100\times5\times50 + 10\times100\times50$ | 75,000 ❌ |

相差 **10 倍**！

### 括号化方案数

$n$ 个矩阵的括号化方案数 = Catalan 数：

$$P(n) = \begin{cases}
1 & n=1,\\
\sum_{k=1}^{n-1} P(k)P(n-k) & n \geq 2
\end{cases} = \Omega(4^n/n^{3/2})$$

指数级！暴力枚举不可行。

### 四步法

#### 步骤 1：最优子结构

设 $A_{i:j} = A_iA_{i+1}\cdots A_j$。

如果最优括号化在 $k$ 处分割（$i \leq k < j$）：
$$A_{i:j} = (A_i\cdots A_k)(A_{k+1}\cdots A_j) = A_{i:k} \cdot A_{k+1:j}$$

那么子链 $A_{i:k}$ 和 $A_{k+1:j}$ 的括号化也必须是最优的。

> **反证法**：如果某个子链可以更优，替换后整体就更优，矛盾。

#### 步骤 2：递归解

设 $m[i,j]$ = 计算 $A_{i:j}$ 所需的最少标量乘法次数。

**基础情况**：$m[i,i] = 0$（单个矩阵，无需乘法）

**递归情况**（$i < j$）：

$$m[i,j] = \min_{i \leq k < j} \{ m[i,k] + m[k+1,j] + p_{i-1}p_kp_j \}$$

其中：
- $m[i,k]$ = 左子链代价
- $m[k+1,j]$ = 右子链代价
- $p_{i-1}p_kp_j$ = 合并两个结果矩阵的代价

#### 步骤 3：计算最优代价

```
MATRIX-CHAIN-ORDER(p, n)
1 let m[1..n, 1..n] and s[1..n-1, 2..n] be new tables
2 for i = 1 to n
3     m[i, i] = 0
4 for l = 2 to n              // l = 子链长度
5     for i = 1 to n-l+1       // i = 起始位置
6         j = i+l-1            // j = 结束位置
7         m[i, j] = ∞
8         for k = i to j-1     // 遍历所有分割点
9             q = m[i, k] + m[k+1, j] + p[i-1]*p[k]*p[j]
10            if q < m[i, j]
11                m[i, j] = q
12                s[i, j] = k  // 记录最优分割点
13 return m and s
```

**关键思路**：按链长度 $l$ 从小到大计算，保证计算 $m[i,j]$ 时所需的子问题 $m[i,k]$ 和 $m[k+1,j]$ 已经算好。

#### 步骤 4：构造最优括号化方案

```
PRINT-OPTIMAL-PARENS(s, i, j)
1 if i == j
2     print "A"i
3 else
4     print "("
5     PRINT-OPTIMAL-PARENS(s, i, s[i, j])
6     PRINT-OPTIMAL-PARENS(s, s[i, j]+1, j)
7     print ")"
```

调用 `PRINT-OPTIMAL-PARENS(s, 1, n)` 输出完整方案。

### 手算演示

设 $p = [5, 4, 6, 2, 7]$（$n=4$，即 $A_1(5\times4), A_2(4\times6), A_3(6\times2), A_4(2\times7)$）

**$l=1$**（初始化）：
$m[1,1]=0, m[2,2]=0, m[3,3]=0, m[4,4]=0$

**$l=2$**：
| $m[i,j]$ | 计算过程 | 结果 | $s[i,j]$ |
|:---:|:---:|---:|:---:|
| $m[1,2]$ | $k=1$: $0+0+5\cdot4\cdot6$ | **120** | 1 |
| $m[2,3]$ | $k=2$: $0+0+4\cdot6\cdot2$ | **48** | 2 |
| $m[3,4]$ | $k=3$: $0+0+6\cdot2\cdot7$ | **84** | 3 |

**$l=3$**：
| $m[i,j]$ | $k$ 取值 | 结果 |
|:---:|:---|---:|
| $m[1,3]$ | $k=1$: $0+48+5\cdot4\cdot2=88$ | **88** |
| | $k=2$: $120+0+5\cdot6\cdot2=180$ | |
| $m[2,4]$ | $k=2$: $0+84+4\cdot6\cdot7=252$ | |
| | $k=3$: $48+0+4\cdot2\cdot7=104$ | **104** |

**$l=4$**：
| $m[i,j]$ | $k$ 取值 | 结果 |
|:---:|:---|---:|
| $m[1,4]$ | $k=1$: $0+104+5\cdot4\cdot7=244$ | |
| | $k=2$: $120+84+5\cdot6\cdot7=414$ | |
| | $k=3$: $88+0+5\cdot2\cdot7=158$ | **158** |

**最优代价**：158 次标量乘法

**最优括号化**：从 $s[1,4]=3$ 回溯 → $(A_1(A_2A_3))A_4$

### 复杂度总结

| 指标 | 值 |
|:---|:---:|
| 子问题个数 | $\Theta(n^2)$ |
| 每个子问题决策数 | $O(n)$ |
| **总时间复杂度** | **$\Theta(n^3)$** |
| **空间复杂度** | **$\Theta(n^2)$**（$m$ 表和 $s$ 表） |

## 14.4 最长公共子序列（Longest Common Subsequence, LCS）

### 什么是子序列？

给定序列 $X = \langle A,B,C,B,D,A,B \rangle$，子序列是从中删除**零个或多个元素**（不要求连续）后得到的序列。

示例：
- $\langle A,B,D \rangle$ ✅（删除 C, C, B, A）
- $\langle B,C,B,A \rangle$ ✅（删除 A, D）
- $\langle A,D,C \rangle$ ❌（违背原始顺序）

### 问题定义

给定两个序列 $X = \langle x_1, x_2, \ldots, x_m \rangle$ 和 $Y = \langle y_1, y_2, \ldots, y_n \rangle$，求一个**长度最大**的公共子序列。

**示例**：
- $X = \langle A,B,C,B,D,A,B \rangle$
- $Y = \langle B,D,C,A,B,A \rangle$
- LCS = $\langle B,C,B,A \rangle$ 或 $\langle B,D,A,B \rangle$（长度 4）

### 最优子结构

设 $Z = \langle z_1, \ldots, z_k \rangle$ 是 $X$ 和 $Y$ 的任意一个 LCS：

| 条件 | 结论 |
|:---|:---|
| $x_m = y_n$ | $z_k = x_m = y_n$，且 $Z_{k-1}$ 是 $X_{m-1}$ 和 $Y_{n-1}$ 的 LCS |
| $x_m \neq y_n$ 且 $z_k \neq x_m$ | $Z$ 是 $X_{m-1}$ 和 $Y$ 的 LCS |
| $x_m \neq y_n$ 且 $z_k \neq y_n$ | $Z$ 是 $X$ 和 $Y_{n-1}$ 的 LCS |

**直观理解**：比较两个序列的最后一个字符
- **相等** → 找到了一个共同字符，双方各退一格
- **不等** → 总得丢掉一个"拖后腿"的，两种都试一下取最优

### 递归解

设 $c[i,j]$ = $X_i$（前 $i$ 个字符）和 $Y_j$（前 $j$ 个字符）的 LCS 长度。

$$c[i,j] = \begin{cases}
0 & \text{若 } i=0 \text{ 或 } j=0, \\
c[i-1,j-1] + 1 & \text{若 } i,j>0 \text{ 且 } x_i = y_j, \\
\max(c[i-1,j], c[i,j-1]) & \text{若 } i,j>0 \text{ 且 } x_i \neq y_j.
\end{cases}$$

### 计算 LCS 长度

```
LCS-LENGTH(X, Y, m, n)
1 let b[1..m, 1..n] and c[0..m, 0..n] be new tables
2 for i = 1 to m
3     c[i, 0] = 0
4 for j = 0 to n
5     c[0, j] = 0
6 for i = 1 to m              // 逐行填充
7     for j = 1 to n
8         if x_i == y_j
9             c[i, j] = c[i-1, j-1] + 1
10            b[i, j] = "↖"      // 来自左上角 → 匹配
11        elseif c[i-1, j] ≥ c[i, j-1]
12            c[i, j] = c[i-1, j]
13            b[i, j] = "↑"      // 来自上方
14        else
15            c[i, j] = c[i, j-1]
16            b[i, j] = "←"      // 来自左方
17 return c and b
```

### 构造 LCS

```
PRINT-LCS(b, X, i, j)
1 if i == 0 or j == 0
2     return
3 if b[i, j] == "↖"
4     PRINT-LCS(b, X, i-1, j-1)
5     print x_i                  // 只有"↖"才输出
6 elseif b[i, j] == "↑"
7     PRINT-LCS(b, X, i-1, j)   // 往上走
8 else
9     PRINT-LCS(b, X, i, j-1)   // 往左走
```

调用 `PRINT-LCS(b, X, m, n)` 输出完整的 LCS。

### 手算演示

$X = \langle A,B,C,B \rangle$, $Y = \langle B,D,C,A \rangle$

| $c[i,j]$ | $j=0$ | $j=1\;B$ | $j=2\;D$ | $j=3\;C$ | $j=4\;A$ |
|:---:|:---:|:---:|:---:|:---:|:---:|
| **$i=0$** | 0 | 0 | 0 | 0 | 0 |
| **$i=1\;A$** | 0 | 0 | 0 | 0 | **1** ↖ |
| **$i=2\;B$** | 0 | **1** ↖ | 1 ← | 1 ← | 1 ← |
| **$i=3\;C$** | 0 | 1 ↑ | 1 ↑ | **2** ↖ | 2 ← |
| **$i=4\;B$** | 0 | 1 ↑ | 1 ↑ | 2 ↑ | **2** ↑ |

**回溯路径**：
```
c[4,4]=2, ↑ → c[3,4]=2, ← → c[3,3]=2, ↖ → 输出 C → c[2,2]=1, ↖ → 输出 B → c[1,1]=0
```

**LCS = $\langle B, C \rangle$** ✅

### 复杂度总结

| 指标 | 值 |
|:---|:---:|
| 子问题个数 | $\Theta(mn)$ |
| 每子问题决策 | $O(1)$（三种情况取其一） |
| **总时间复杂度** | **$\Theta(mn)$** |
| **空间复杂度** | **$\Theta(mn)$**（$c$ 表 + $b$ 表） |

### 对比：矩阵链乘 vs LCS

| 特性 | 矩阵链乘 | LCS |
|:---|:---|:---|
| 输入 | 1 个序列（$n$ 个矩阵） | 2 个序列（$X$, $Y$） |
| 子问题维度 | 区间 $[i,j]$ | 前缀对 $(i,j)$ |
| 填表方向 | 按长度 $l$ 递增 | 按行/列顺序 |
| 递推操作 | $\min\{...+p_{i-1}p_kp_j\}$ | $\max\{...,\;...+1\}$ |
| 时间复杂度 | $\Theta(n^3)$ | $\Theta(mn)$ |
| 经典应用 | 矩阵乘法优化 | DNA 比对、diff 工具 |

## 14.5 最优二叉搜索树（Optimal Binary Search Tree, OBST）

### 问题背景

假设你正在写一个**英译法**程序，需要反复查词典。每个单词有不同的搜索频率：

| 单词 | 频率 |
|:---|:---:|
| "a" | 非常高 |
| "the" | 高 |
| "algorithm" | 低 |
| "zythum" | 极低 |

如果 "zythum" 在根节点，每次查 "a" 都得走到很深的叶子——效率极低！

**目标**：把频率高的单词放得靠近根，频率低的放远些，使得**期望搜索代价最小**。

### 与普通 BST 的区别

| 特性 | 普通 BST（平衡） | 最优 BST |
|:---|:---|:---|
| 目标 | 高度最小化 | **期望搜索代价最小化** |
| 依据 | 只考虑键的大小 | 考虑键的**访问概率** |
| 结果 | 固定结构 | 根据概率调整结构 |

### 形式化定义

**输入**：
- $n$ 个键：$k_1 < k_2 < \ldots < k_n$
- 成功搜索概率：$p_i$ = 搜索 $k_i$ 的概率
- $n+1$ 个虚拟键：$d_0, d_1, \ldots, d_n$（表示不在 BST 中的值）
  - $d_0$：所有小于 $k_1$ 的值
  - $d_i$：介于 $k_i$ 和 $k_{i+1}$ 之间的值
  - $d_n$：所有大于 $k_n$ 的值
- 失败搜索概率：$q_i$ = 搜索落在 $d_i$ 区间的概率

**概率归一化**：

$$\sum_{i=1}^n p_i + \sum_{i=0}^n q_i = 1$$

**期望搜索代价**：

$$E[\text{搜索代价}] = 1 + \sum_{i=1}^n \text{depth}(k_i) \cdot p_i + \sum_{i=0}^n \text{depth}(d_i) \cdot q_i$$

其中 $\text{depth}$ 是节点深度（根深度为 0），加 1 是因为比较次数 = 深度 + 1。

### 最优子结构

> 一棵最优 BST 的任意**子树**也必须是**最优的**。

设子问题为：构造包含连续键 $k_i, \ldots, k_j$（以及对应的虚拟键 $d_{i-1}, \ldots, d_j$）的最优 BST。

### 递归解

设 $e[i,j]$ = 包含键 $k_i, \ldots, k_j$ 的最优 BST 的期望搜索代价。

**基础情况**（$j = i-1$，即空子树，只有虚拟键 $d_{i-1}$）：

$$e[i, i-1] = q_{i-1}$$

> 树中只有一个虚拟键节点 $d_{i-1}$，深度为 0，搜索它需要 1 次比较，代价 = $1 \times q_{i-1} = q_{i-1}$。

**递归情况**（$j \geq i$）：选择根 $k_r$（$i \leq r \leq j$）

$$e[i, j] = \min_{i \leq r \leq j} \big\{ e[i, r-1] + e[r+1, j] + w(i, j) \big\}$$

其中：

$$w(i, j) = \sum_{l=i}^j p_l + \sum_{l=i-1}^j q_l$$

### $w(i,j)$ 的直观理解：搬家涨房租 🏠

**核心思想**：当一棵子树被挂到某个根节点下面，子树中所有节点的深度都 +1。

想象一栋公寓楼（子树）里的所有住户（节点）整体**往上挪一层**：
- 每个住户多付的租金 = 该住户的概率
- 总租金上涨额 = **所有住户概率之和**

$$w(i,j) = \text{子树中所有节点（实际键 + 虚拟键）的概率总和}$$

**推导过程**：

左子树 $T_L$ 的**原始**期望代价：$e[i, r-1]$

当 $T_L$ 作为 $k_r$ 的左子树时，所有节点深度 +1：

$$\begin{aligned}
\text{新代价} &= \sum (\text{depth}_{\text{old}} + 1) \times \text{prob} \\
&= \sum \text{depth}_{\text{old}} \times \text{prob} + \sum 1 \times \text{prob} \\
&= e[i, r-1] + w(i, r-1)
\end{aligned}$$

同样，右子树新代价 = $e[r+1, j] + w(r+1, j)$，根节点代价 = $p_r$。

合并三个 $w$ 项：

$$w(i, r-1) + w(r+1, j) + p_r = w(i, j)$$

所以最终：

$$e[i,j] = e[i, r-1] + e[r+1, j] + w(i, j)$$

> **一句话**：最优代价 = 左子树最优 + 右子树最优 + 左右子树下移一层的涨价 + 根节点自己的代价

### 关于 $j = i-1$ 的详细解释

**问题**：为什么 $e[i, i-1] = q_{i-1}$？

**含义**：子问题的上界 $j$ 比下界 $i$ 小 1，说明**没有任何实际键**，只有虚拟键 $d_{i-1}$。

**示例**：

| $e[i,j]$ | 含义 | 值 |
|:---|:---|:---:|
| $e[1,0]$ | 只有 $d_0$（小于 $k_1$ 的所有值） | $q_0$ |
| $e[3,2]$ | 只有 $d_2$（$k_2$ 和 $k_3$ 之间的值） | $q_2$ |
| $e[6,5]$ | 只有 $d_5$（大于 $k_5$ 的所有值） | $q_5$ |

**为什么伪代码中 $e$ 的索引范围是 $[1..n+1, 0..n]$？**

因为尝试根 $r$ 时，左右子树可能为空：
- $r=1$：左子树为空 → $e[1,0] = q_0$
- $r=n$：右子树为空 → $e[n+1, n] = q_n$

所以需要 $e$ 能访问到 $e[1,0]$ 和 $e[n+1,n]$。

### 计算最优代价

```
OPTIMAL-BST(p, q, n)
1 let e[1..n+1, 0..n], w[1..n+1, 0..n],
       and root[1..n, 1..n] be new tables
2 for i = 1 to n+1
3     e[i, i-1] = q_{i-1}           // 初始化空子树
4     w[i, i-1] = q_{i-1}
5 for l = 1 to n                     // l = 子问题长度
6     for i = 1 to n-l+1
7         j = i+l-1
8         e[i, j] = ∞
9         w[i, j] = w[i, j-1] + p_j + q_j   // 前缀和递推，O(1) 计算 w(i,j)
10        for r = i to j                     // 尝试每个可能的根
11            t = e[i, r-1] + e[r+1, j] + w[i, j]
12            if t < e[i, j]
13                e[i, j] = t
14                root[i, j] = r
15 return e and root
```

**$w(i,j)$ 的递推计算**（第 9 行）：
- $w[i,j] = w[i,j-1] + p_j + q_j$
- 利用前缀和思想，$O(1)$ 时间得到任意 $w(i,j)$
- 否则每次累加 $O(n)$，总时间退化到 $O(n^4)$

### 复杂度总结

| 指标 | 值 |
|:---|:---:|
| 子问题个数 | $\Theta(n^2)$ |
| 每子问题决策数 | $O(n)$ |
| **总时间复杂度** | **$\Theta(n^3)$** |
| **优化后** | **$\Theta(n^2)$**（Knuth 优化） |
| **空间复杂度** | **$\Theta(n^2)$**（$e$, $w$, $root$ 表） |

### LCS vs OBST 对比

| 特性 | LCS | OBST |
|:---|:---|:---|
| 子问题形状 | 前缀对 $(i,j)$ | 连续区间 $[i,j]$ |
| 分割方式 | 丢弃字符 | 选根分割 |
| 基础情况 | $i=0$ 或 $j=0$ | $j = i-1$ |
| 递推 | $\max\{...,...+1\}$ | $\min\{...+w(i,j)\}$ |
| 额外代价项 | 无 | $w(i,j)$（深度增加） |
| 时间复杂度 | $\Theta(mn)$ | $\Theta(n^3)$ |
| 应用 | DNA 比对、diff | 词典、编码树 |
