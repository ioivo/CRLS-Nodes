
# 12.1-3

## 题目要求
设计一个非递归算法实现二叉搜索树（BST）的中序遍历。
1. **基础版**：使用栈（Stack）作为辅助数据结构。
2. **进阶版**：不使用栈，空间复杂度为 $O(1)$。假设可以测试两个指针的相等性（通常暗示存在父节点指针 `parent`）。

---

## 方案一：辅助栈法（空间复杂度 $O(n)$）

### 核心思路
中序遍历的顺序是 **左 -> 根 -> 右**。非递归实现的关键在于：利用栈来模拟递归时的“回溯”过程。
1. 将当前节点及其所有左侧后代压入栈中。
2. 当左侧为空时，弹出栈顶元素（即当前最左节点），进行访问。
3. 转向该节点的右子树，重复上述过程。

### C++ 实现
```cpp
void inorderTraversalWithStack(TreeNode* root) {
    std::stack<TreeNode*> s;
    TreeNode* curr = root;

    while (curr != nullptr || !s.empty()) {
        // 1. 尽可能向左走
        while (curr != nullptr) {
            s.push(curr);
            curr = curr->left;
        }
        
        // 2. 弹出并访问
        curr = s.top();
        s.pop();
        std::cout << curr->key << " ";
        
        // 3. 转向右子树
        curr = curr->right;
    }
}
```

## 方案二：父指针法（空间复杂度 $O(1)$）

### 核心思路
这是题目中提到的“优雅解法”。当节点结构中包含 `parent` 指针时，我们可以通过比较“当前节点”与“前驱节点”的位置关系，判断遍历的方向，从而摆脱对栈的依赖。

**方向判定逻辑：**
- **从父节点进入：** 优先向左走；若左子树为空，则访问自己并尝试向右走。
- **从左子树返回：** 说明左侧已处理完，访问自己并尝试向右走。
- **从右子树返回：** 说明当前子树已全部处理完，向上回溯到父节点。

### C++ 实现
```cpp
void inorderTraversalWithParent(TreeNode* root) {
    TreeNode* curr = root;
    TreeNode* prev = nullptr;
    TreeNode* next = nullptr;

    while (curr != nullptr) {
        if (prev == curr->parent) {
            // 情况 A：从上方（父节点）下来
            if (curr->left != nullptr) {
                next = curr->left; // 优先往左走
            } else {
                std::cout << curr->key << " "; // 左边没了，访问自己
                next = (curr->right != nullptr) ? curr->right : curr->parent;
            }
        } else if (prev == curr->left) {
            // 情况 B：从左子树升上来
            std::cout << curr->key << " "; // 左边处理完了，访问自己
            next = (curr->right != nullptr) ? curr->right : curr->parent;
        } else {
            // 情况 C：从右子树升上来
            next = curr->parent; // 右边处理完了，继续往上走
        }

        prev = curr;
        curr = next;
    }
}
```

# BST 节点前驱与后继 (Predecessor & Successor) 总结

在二叉搜索树（BST）中，前驱和后继定义了中序遍历序列中某个节点的相邻关系。

## 1. 核心定义
* **Successor (后继)**：中序遍历中紧跟在 $x$ 之后的节点。即：键值大于 $x$ 的所有节点中最小的一个。
* **Predecessor (前驱)**：中序遍历中排在 $x$ 之前的节点。即：键值小于 $x$ 的所有节点中最大的一个。

---

## 2. 寻找逻辑

### A. 寻找后继 (Successor)
1.  **存在右子树**：后继是其右子树中的**最小值**。
    * *操作*：`node = node->right`，然后一直向左走到底。
2.  **不存在右子树**：后继是其某个祖先。
    * *操作*：向上回溯，直到找到一个节点，它是其父节点的**左孩子**。该父节点即为后继。



### B. 寻找前驱 (Predecessor)
1.  **存在左子树**：前驱是其左子树中的**最大值**。
    * *操作*：`node = node->left`，然后一直向右走到底。
2.  **不存在左子树**：前驱是其某个祖先。
    * *操作*：向上回溯，直到找到一个节点，它是其父节点的**右孩子**。该父节点即为前驱。

---

## 3. 关键性质
* **复杂度**：单次寻找操作的时间复杂度为 $O(h)$，其中 $h$ 是树的高度。
* **全遍历性能**：从最小值开始连续调用 $n-1$ 次 Successor 遍历整棵树，总时间复杂度为 **$\Theta(n)$**（摊还分析：每条边最多被访问两次）。
* **叶子节点性质**：对于叶子节点，其父节点要么是它的前驱，要么是它的后继。
* **双分支节点性质**：如果一个节点有两个孩子，其后继一定没有左孩子，前驱一定没有右孩子。

---

## 4. C++ 代码参考 (带 Parent 指针)

```cpp
// 寻找后继节点
TreeNode* getSuccessor(TreeNode* x) {
    if (x == nullptr) return nullptr;

    // 情况 1: 有右子树
    if (x->right != nullptr) {
        x = x->right;
        while (x->left != nullptr) x = x->left;
        return x;
    }

    // 情况 2: 无右子树，向上找
    TreeNode* y = x->parent;
    while (y != nullptr && x == y->right) {
        x = y;
        y = y->parent;
    }
    return y;
}

// 寻找前驱节点 (对称逻辑)
TreeNode* getPredecessor(TreeNode* x) {
    if (x == nullptr) return nullptr;

    if (x->left != nullptr) {
        x = x->left;
        while (x->right != nullptr) x = x->right;
        return x;
    }

    TreeNode* y = x->parent;
    while (y != nullptr && x == y->left) {
        x = y;
        y = y->parent;
    }
    return y;
}
```

## 5. 常见面试考点
> **Q：如果在没有 `parent` 指针的情况下找后继怎么办？**
>
> **A**：需要从根节点（Root）开始搜索。
> 1. 如果目标值小于当前节点，当前节点可能是后继，记录之并向左找。
> 2. 如果目标值大于当前节点，向右找。
> 3. 直到找到目标节点，若它有右子树，则直接找右子树最小值；否则，之前记录的最后一个“向左转”时的节点就是后继。


## 12.3-6：使用 successor 代替 parent 指针

**题目背景：**
假设二叉搜索树（BST）的节点中没有 `parent` 指针，而是包含了一个指向其后继的 `succ` 指针。如何在 $O(h)$ 的时间复杂度内完成所有的基本操作（其中 $h$ 为树高）？

**核心思路与做法：**

1. **`TREE-SEARCH`（搜索）**：
   - 本身就是自顶向下的查找过程，完全不需要 `parent` 指针，逻辑保持不变，时间复杂度 $O(h)$。

2. **`TREE-INSERT`（插入）**：
   - **查找位置**：在向下寻找插入位置时，手动维护两个指针（当前节点 `x` 和它的父节点 `y`）。
   - **更新后继**：插入新节点 $z$ 后，根据它是左孩子还是右孩子来更新 `succ`：
     - 若 $z$ 是 $y$ 的左孩子：$z$ 的后继就是 $y$（`z.succ = y`），并且需要找到 $z$ 的前驱节点，将其 `succ` 更新为 $z$。
     - 若 $z$ 是 $y$ 的右孩子：$z$ 继承 $y$ 的后继（`z.succ = y.succ`），然后 $y$ 的后继变为 $z$（`y.succ = z`）。

3. **`TREE-DELETE`（删除）**：
   - **辅助函数 `FIND-PARENT(T, x)`**：因为删除了 `parent` 指针，我们需要一个 $O(h)$ 的辅助函数。从根节点往下搜索 `x`，在搜索路径上记录上一个节点，即可找到父节点。
   - **执行删除**：将原始算法中所有调用 `z.p` 的地方，替换为调用 `FIND-PARENT(T, z)`。
   - **修复后继**：删除节点 $z$ 后，必须找到 $z$ 的前驱节点，并将其 `succ` 指针跳过 $z$，直接指向 $z$ 的后继（即 `z.succ`）。
   - **复杂度**：虽然增加了几次 $O(h)$ 的向下搜索操作来寻找父节点和前驱，但由于是常数次调用，总时间复杂度依然维持在 $O(h)$。

## 12.3-7：公平删除策略（解决树的倾斜问题）

**题目背景：**
在标准的 `TREE-DELETE` 操作中，当要删除的节点有两个孩子时，我们总是寻找它的**后继**（右子树的最小值）来替换它。长此以往，这种不对称的操作会导致树的结构向一侧倾斜（通常是左倾），从而增加树的平均高度。

**核心思路与做法：**

1. **固定改为使用“前驱”替换的对称逻辑**：
   如果我们要始终用左子树的最大值（前驱）来替换，`TREE-DELETE` 的逻辑需进行对称性修改：
   - 找到目标节点 $z$ 左子树中的最大值 $y$（即前驱）。
   - 如果 $y$ 不是 $z$ 的直接左孩子，则用 $y$ 的左孩子替换 $y$（此时 $y$ 必定没有右孩子）。
   - 用 $y$ 替换 $z$，并将 $z$ 的右子树连接到 $y$ 的右侧。

2. **实现“公平策略 (Fair Strategy)”**：
   为了防止树向任何一侧倾斜，我们可以引入**随机化**机制，使其在统计学上保持平衡。
   - **具体做法**：在 `TREE-DELETE` 中，当判断出目标节点 $z$ 同时拥有左右子树时，调用一个随机数生成器（如抛硬币 `RAND() % 2`）。
   - 如果结果为 0：执行标准的后继替换逻辑（用右子树的最小值替换）。
   - 如果结果为 1：执行上述修改后的前驱替换逻辑（用左子树的最大值替换）。

# 12-1: 等键二叉搜索树

## 背景：问题的核心
标准的 `TREE-INSERT` 在处理 `z.key == x.key` 时，通常固定向右（或向左）走。如果你输入 $n$ 个完全相同的数字，这会导致算法在每一层都只往一个方向跑，最后得到一个高度为 $n$ 的单链表。

---

## 策略分析与做法

### (a) 标准策略（无脑走一边）
* **做法**：若 `z.key == x.key`，永远执行 `x = x.right`。
* **树的形态**：形成一根向右倾斜的**单链表**（高度 $h = n$）。
* **性能**：
    * 第 $i$ 次插入需要比较 $i-1$ 次。
    * 总时间 = $0 + 1 + 2 + \dots + (n-1)$。
    * **复杂度**：$$\Theta(n^2)$$
> **毒舌评价**：这是最偷懒的做法，在数据重复率高的情况下简直是性能灾难。



### (b) 旗帜交替策略（左右横跳）
* **做法**：每个节点 $x$ 存一个布尔值 $x.b$。每当有等键节点经过 $x$ 时：
    * 若 $x.b = \text{FALSE}$，去左边，并将 $x.b$ 设为 $\text{TRUE}$。
    * 若 $x.b = \text{TRUE}$，去右边，并将 $x.b$ 设为 $\text{FALSE}$。
* **树的形态**：等键节点被均匀分发到左右子树，形成一棵**近似完全二叉树**。
* **性能**：
    * 树高 $h = \Theta(\log n)$。
    * **复杂度**：$$\Theta(n \log n)$$

### (c) 链表聚合策略（原地扎堆）
* **做法**：每个节点不只存一个值，而是维护一个**链表**。遇到等键节点，直接挂在对应键值节点的链表里，不往深处走。
* **树的形态**：如果 $n$ 个键全相等，整棵树**只有一个节点**，节点下面挂着长长的链表。
* **性能**：
    * 插入时只需找到根节点（$O(1)$），然后插入链表（$O(1)$）。
    * **复杂度**：$$\Theta(n)$$
> **注意**：虽然速度最快，但在混合数据下会增加节点结构的复杂度和内存开销。

### (d) 随机化策略（听天由命）
* **做法**：遇到等键时，以 $1/2$ 的概率随机选左或选右。
* **性能分析**：
    * **最坏情况**：依然是 $$\Theta(n^2)$$（虽然概率比你中彩票还低，但理论上存在）。
    * **期望运行时间**：等同于随机构建的 BST，期望高度为 $O(\log n)$。
    * **期望复杂度**：$$\Theta(n \log n)$$

## 总结表：$n$ 个等键插入性能对比

| 策略 | 实现难度 | 空间开销 | 树高度 | 时间复杂度 | 推荐场景 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **(a) 固定方向** | 极低 | 无额外 | $n$ | $\Theta(n^2)$ | 仅限学术反例 |
| **(b) 布尔标志** | 低 | 1 bit/节点 | $\log n$ | $\Theta(n \log n)$ | **最推荐**，性能稳定 |
| **(c) 链表节点** | 中 | 链表指针 | $1$ (等键时) | $\Theta(n)$ | 存在大量重复键时 |
| **(d) 随机化** | 低 | 无额外 | $\log n$ (期望) | $\Theta(n \log n)$ | 追求极致代码简洁 |
